* usando o git pra ver as alteracoes no respositorio
  Bom, depois que a gente clonou nosso repositório, entramos dentro
  dele e criamos o nosso projeto do maven. Obviamente o nosso
  repositório teve alterações. A vantagem de utilizar esse repositório
  é que o que a gente fizer dentro dele a gente pode subir tudo pro
  github e ainda tem muitas outras vantagens que não pra escrever tudo
  aqui.
  Por agora queremos ver o que mudou no nosso repositório e também
  queremos subir pro github.
  Então, vendo o que mudou... é sinomimo de conhecer o status da nossa
  cópia local do repositório ...
  Lembrando que esse comando vc tem que dar dentro do diretório do seu respositório.

*** colocando a dependencia do jdbc no nosso classpah
O professores vai te explicar o que são essas dependencias e o que é
classpath.
Veja que já temos as classes que necessárias pra criar a conexão (Jdbcutil.java) e pra
usar  a conexão (ClienteJdbcDao.java).
Além disso nosso botão "SAVLAR" também já aciona direitinho essas
classes pra salvar um cliente.
Mas se vc rodar o projeto vai dar erro, porque vc tem que colocar o
conector do banco de dados no classpath da sua aplicação.
Pra fazer isso vc deve colocar o seguinte código na sessão
"<dependencies>' do seu pom.xml...

#+BEGIN_SRC xml
    <!-- https://mvnrepository.com/artifact/com.microsoft.sqlserver/mssql-jdbc -->
    <dependency>
      <groupId>com.microsoft.sqlserver</groupId>
      <artifactId>mssql-jdbc</artifactId>
      <version>6.1.0.jre8</version>
    </dependency>
    <!-- https://mvnrepository.com/artifact/mysql/mysql-connector-java -->
    <dependency>
      <groupId>mysql</groupId>
      <artifactId>mysql-connector-java</artifactId>
      <version>5.1.6</version>
    </dependency>    
#+END_SRC
Isso importa pro seu projeto os conectores fazendo o donwload deles
automaticamente e colocando no seu projeto. Veja que tem o conector do
sqlserver e do mysql.

Só que não basta só fazer o download dessas dependencias, você quer
que elas acompanhem o executável do seu projeto. Vc vai querer que seu
jar gerado pela aplicação seja executável nao vai? Claro... não
estamos desenvolvendo uma api, é um projeto que a gente quer que o
usuário execute, então vc realmente quer que o seu jar seja
executável.
Isso não é totalmente de graça.Vc tem que colocar os seguintes trechos
no seu pom.xml.

#+BEGIN_SRC xml
      <plugin>
        <groupId>org.apache.maven.plugins</groupId>
        <artifactId>maven-jar-plugin</artifactId>
        <version>3.0.2</version>
	<configuration>
	  <archive>
	    <manifest>
	      <addClasspath>true</addClasspath>
              <mainClass>br.edu.etec.lojainformatica.App</mainClass>
	      <classpathPrefix>${project.build.outputDirectory}.lib</classpathPrefix>
	    </manifest>
	  </archive>
        </configuration>                                
      </plugin>
#+END_SRC

Esse trecho acima faz com que o jar que o maven cria tenha o seu
arquivo Manifest.MF alterado indicando qual é  sua classe que tem o
método main que vc quer que execute quanto o usuário executar o seu
jar e também a configuração do classpath pra que seu projeto gerado
consiga enxergar os jars de que ele dependente pra executar, no caso
os jars conectores do banco de dados.


O trecho precisa estar no seu pom.xml pra que os jars do seu projeto
seja copiado pra um local que o seu jar executável possa
conhecê-los. Leia o xml que vai fazer sentido.. vc vai ver que é um
plugin do maven pra lidar com dependencias e também que está escrito
copy lá deixando claro o que ele vai fazer pra gente.
#+BEGIN_SRC xml
      <plugin>
	<artifactId>maven-dependency-plugin</artifactId>
	<executions>
	  <execution>
	    <phase>package</phase>
	    <goals>
	      <goal>copy-dependencies</goal>
	    </goals>
	    <configuration>
	      <outputDirectory>${project.build.outputDirectory}.lib</outputDirectory>
	    </configuration>
	  </execution>
	</executions>
      </plugin>
#+END_SRC

Já essa parte de baixo é pra gente rodar o nosso projeto usando o
comando maven. Nem sempre eu uso, mas se vcs quizerem usar é so dar um
mvn exec:java br.edu.etec.lojainformatica.App que o projeto
executa. Isso seria muito útil caso a gente já não tivesse resolvido a
questão de copiar as dependencias pro nosso pacote (jar)
executável. Agora que a gente resolveu o problema das dependencias a
gente pode fazer o seguinte pra rodar o projeto: java -jar target/jardonossoprojeto.jar
