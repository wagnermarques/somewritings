TCM determinístico que
utilize a comparação em pares precisaria de 2 n execuções para encontrar o subconjunto
de teste ideal. Considerando sua complexidade assintótica O(2 n ), mesmo para valores
moderados de n, 2 n tende ao infinito.
Em outras palavras, a TCM é um problema NP-Hard (set cover problem), pois as
possibilidades de busca são iguais à quantidade de todas as combinações entre os dados do
conjunto de testes (HOCHBA, 1997; DEVORE, 2012).

A TCM otimiza conjuntos de
testes pré-existentes e pode ser guiada por diferentes perspectivas do software sob teste,
conforme descrito a seguir.
A seleção com base no domínio do software aborda um conjunto de problemas
inerentes ao domínio aplicacional do SUT, os quais as técnicas de TCM
devem solucionar.
